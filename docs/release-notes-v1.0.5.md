# codexU v1.0.5

本次 patch 版本在保留完整视觉体验的前提下继续降低资源占用：额度区域会根据 Codex 实际返回的窗口自适应单环或双环，粒子仅在真正可见且需要交互时渲染，后台刷新与 Claude Code transcript 缓存也进一步收紧。

## 主要更新

- Codex 仅返回 7 天额度时，主界面、菜单栏和 Runtime 浮窗自动使用单环、单进度条与居中百分比；返回双窗口时恢复双额度布局，明确无额度限制时显示无限制状态。
- 失败、畸形、未知、重复或部分额度响应不再触发布局误判，而是保留最近一次可信拓扑并标记陈旧状态。
- 恢复完整环形粒子效果并约束在进度环描边内；默认仅在主窗口可见、置前且聚焦时渲染，新增“省电”模式，仅在悬停额度环时显示动效。
- 窗口失焦、关闭、最小化、被遮挡或位于其他 Space 时停止粒子；低电量模式、温控压力和“减少动态效果”也会自动降载。
- 菜单栏单额度布局将百分比居中显示，并根据填充色与灰色轨道自动切换文字颜色；重置倒计时改为 `↻ 5d` 等更清晰的紧凑表达。
- 任务看板只在相关视图活跃时保持前台刷新，后台使用更低频且可合并的定时器。
- Claude Code transcript 缓存升级为纳秒级文件指纹，兼容迁移旧缓存、清理已删除文件记录并报告写入失败。

## 验证

- 通过统计时区、额度拓扑、状态栏布局、粒子生命周期、更新检查和解析器自测。
- 通过单额度/双额度、异常响应连续性、粒子可见性与 Claude Code 缓存迁移回归测试。
- 通过 Apple Silicon 与 Intel 双架构 DMG、checksum、挂载、Mach-O 架构和 codesign 验证。
