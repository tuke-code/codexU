# codexU Design System

本文档定义 codexU 的长期视觉与交互规范。它不是某个版本或某次改版的复盘，而是后续所有界面迭代都应遵守的设计边界。

## 设计立场

codexU 是一个 macOS 菜单栏和桌面工作型工具。它应该安静、清晰、可信，优先服务高频扫视和快速判断，而不是制造展示感。

- 像原生工具，不像网页落地页。
- 以 Liquid Glass 为核心视觉范式，用半透明材质、柔和边界和环境融合建立空间感。
- 信息密度高，但层级清楚。
- 颜色用于语义和强调，不用于装饰堆叠。
- 数据口径诚实，不把估算包装成精确事实。
- 默认保护隐私，不展示用户内容正文。

## 信息原则

- 先展示用户要做判断所需的信息，再展示解释性细节。
- 聚合信息优先于原始日志，趋势和排行优先于长文本。
- 文案使用用户语言，不暴露内部字段名。
- 不确定、回退、估算的数据必须明确标注。
- 桌面首屏不展示 prompt、回复正文、tool arguments、auth、raw logs 或附件内容。

## 视觉语言

### 材质

- 界面基调是 Liquid Glass：透明、轻盈、带环境感，但始终以内容可读为前提。
- 窗口使用系统玻璃建立环境融合与空间层级；内部大分组使用静态光学 surface 延续玻璃视觉。
- 每个窗口最多保留一层实时系统玻璃；内部大分组和卡片使用静态半透明填充与细描边，边界清楚但不厚重。
- 玻璃层级必须克制；禁止叠加多个实时背景采样层。
- 高光、描边、阴影只用于表达层级和交互状态，不作为装饰元素堆叠。
- 不使用大面积渐变、装饰性光斑、背景 orb 或过度阴影。
- 复杂壁纸、浅色模式、深色模式下都必须保持文字可读。

### 色彩

色彩分为四类，不跨职责复用：

| 类别 | 用途 |
| --- | --- |
| Accent | 当前配色的主要数据强调、主操作与选中态 |
| Status | 成功、警告、错误、等待、完成等状态 |
| Data | 图表、token 拆分、趋势强弱 |
| Surface | 窗口、分组、卡片、轨道、分割线 |

规则：

- 默认配色保持蓝紫体系；其他内置或社区配色只能通过受校验的 Palette Package 提供 Accent、Quota、Data、Selection 与 Ornament 语义。
- Surface、Text、Control、Motion 和 Status 是产品固定设计体系，不允许配色包覆盖。切换配色不能改变危险、成功、警告等状态含义，也不能重写窗口层级。
- 绿色只表示成功、完成或健康状态，不作为普通数据主色。
- 橙色只表示注意、进行中或输出类数据，不作为危险色。
- 红色只表示错误、危险或不可继续。
- 图表色阶使用同一语义色族的明度变化，避免彩虹色。
- 进度填充使用同一蓝紫色族时，从浅色起点平滑过渡到深色终点。
- 额度环轨道必须先按半线宽内缩再描边；进度弧与粒子共享同一中心线，粒子的径向外沿不得越出有效进度描边。单额度使用外环尺寸，双额度才使用同心双环。
- 新增可变颜色必须进入配色包 Token 契约并说明语义角色；新增固定状态或表面色必须进入 `FixedVisualPalette`，不得在组件局部硬编码。
- 持续动效只用于表达数据流向或活动性；双环粒子保留完整的独立速度、相位和点状拖尾，但必须按状态挂载：默认模式仅在主窗口位于最前且聚焦时创建，省电模式还要求指针悬停在环带上。窗口失焦、隐藏、关闭、最小化、完全遮挡、切换到非活动桌面，或系统进入低电量、温控压力及“减少动态效果”状态时，立即移除动画和粒子层，不得仅暂停或隐藏。

### 字体

- 使用系统字体和系统文本层级。
- 关键数值使用圆角数字风格和等宽数字，减少刷新跳动。
- 小字号文字使用 `.secondary` 或 `.tertiary` 语义色；时间、项目、状态等必要信息不得使用低对比度 `.tertiary`。
- 不使用随窗口宽度连续缩放的字体。
- 标题、数值、说明、标签必须有稳定层级，不用同一字号硬撑所有信息。

## 布局原则

- 所有区域遵循清晰的层级：窗口、分组、卡片、行、控件。
- 主窗口以 820pt 紧凑布局为设计母版，允许在 820–1280pt 范围内调整宽度；增加宽度只为现有卡片、任务列、图表和排行分配更多空间，不改变组件顺序、行列关系或固定间距。
- 主窗口高度允许随屏幕可用区域调整；内容不足时保持顶部对齐，内容超出时只在主内容区滚动。
- 窗口尺寸变化不触发布局断点，也不随宽度连续缩放字体、图标、间距或圆角；“羊毛进度”始终位于三个 Token 统计卡片下方。
- 用户调整后的主窗口尺寸与位置应由系统恢复；首次启动仍使用紧凑布局。
- 卡片之间对齐优先于单卡内容“看起来刚好”。
- 并列卡片必须共享顶部基线；同一行卡片需要等高时必须显式约束高度。
- 标题栏高度、图标尺寸、右侧控件高度应稳定，不随文案或数据刷新抖动。
- 卡片内边距、行距、圆角、列表间距使用统一设计 token，不在局部组件随意新增数值。
- 不把卡片嵌套在卡片里；重复列表项可以使用轻量行背景。
- 文本不得溢出容器；长项目名、路径、数字都必须有截断或缩放策略。

## 组件规范

### Header

- logo、产品名、账户状态、全局操作分区明确。
- 全局操作使用 icon button，必要时提供 tooltip。
- 状态 pill 使用中性样式，只有异常或风险时引入状态色。

### Tabs

- tab 是内容区标题的一部分，位于内容区左侧。
- tab 必须包含图标和短文本。
- 当前 tab 清晰可见，但不使用强品牌色块压过内容。
- tab 行右侧用于放当前视图 summary，不放主要操作。

### Cards

- 卡片标题由图标、标题和可选右侧控件组成。
- 右侧控件只承载筛选、口径、轻量状态，不承载长说明。
- 说明性长文放 tooltip 或文档，不塞进标题栏。
- 卡片底部对齐比局部内容紧凑更重要。
- 主界面可用重置明细常驻最多两条，不增加“其余次数”或“悬停查看”提示行；只要存在返回明细，无论总次数是否超过三次，悬停常驻明细都使用用量趋势同款 tooltip 展示完整返回明细与缺失说明。tooltip 使用清晰可辨的半透明表面并紧随鼠标位置，必须覆盖相邻卡片，不得被后绘制的内容遮挡或截断鼠标追踪。

### Task Board

- 任务列名、状态文案和颜色必须匹配当前 Runtime 的本地事实源；活动时间只能表达最近活动，归档不能表达成功，未知状态必须明确降级。
- 卡片信息优先级固定为标题、工作区或周期、事实时间、可信状态；技术编号与累计 Token 只能作为辅助信息，缺失时不显示伪造占位值。
- 定时任务优先展示周期和可确定的下次运行时间；配置更新时间不得放在运行时间位置，规则不完整时不得猜测。
- 只有存在真实操作的卡片才能呈现 Button、hover、手型指针和焦点反馈；静态卡片、标题栏占位和无行为图标不得伪装成控件。
- 状态必须同时使用文字和 SF Symbol；tooltip 与可访问性文案可以解释来源口径，但不得展示 prompt、回复、命令或 raw logs。

### Lists

- 列表项结构保持稳定：名称、辅助信息、主数值、次数值、相对条形。
- 主数值右对齐，名称左对齐。
- 行间距稳定，刷新时不产生明显跳动。
- 进度条只表达相对关系，不替代精确数值。

### Charts

- 图表必须服务一个明确问题：趋势、分布、排行或构成。
- 轴、图例、tooltip 的位置保持稳定。
- 图例色块和图中标记大小保持一致。
- hover tooltip 只显示必要聚合数据，不显示敏感原文。
- 空值、零值、未来日期和缺失数据必须有不同语义，不用同一种强提示。

### Menu Bar Status

- 状态栏使用简约、经典、丰富三档稳定密度；模式控制表达方式，指标选择控制内容，两者不能通过排列组合复制多套业务逻辑。
- 状态栏图片背景保持透明，不自绘常驻灰色胶囊；hover/pressed 背景由 macOS 状态栏负责。
- 简约模式不显示 Runtime Logo；单额度时使用加粗单环，双额度时使用同心双环。双环必须共用圆心并保留透明间隙，不得重叠或粘连。经典模式的独立微型环内只保留百分比数字；丰富模式在单额度时使用数字居中的高对比胶囊进度条，双额度时使用两条带标签的线性进度条。
- Palette 身份与外环/内环几何位置必须分离。5h 始终使用 `quota.primary`，7d 始终使用 `quota.secondary`，不能因单额度或排列位置改变。Palette Package v1 没有 monthly 专用角色：monthly 默认回退到 `quota.secondary`；仅在 `7d + monthly` 且不存在 5h 时，monthly 使用空闲的 `quota.primary`，7d 仍保持 `quota.secondary`。颜色标识额度窗口，不随已用/剩余口径或百分比切换。状态栏只使用 L0 Token 渐变，不加载 SVG 纹样。
- “已用量 / 剩余量”必须同时作用于环、条、数字、tooltip 和可访问性文案；已用按顺时针/左到右绘制，剩余按逆时针/右到左绘制，状态栏内不额外增加口径文字。
- 经典和丰富模式的 Runtime 图标使用 18pt 常规状态栏尺寸；简约模式外环/内环分别使用不低于 2.5pt/2.2pt 的线宽，并保持至少 0.75pt 的环间径向间隙。
- 状态栏 Runtime 图标使用从原始品牌 Logo 确定性派生的透明单色模板，不直接复用白底彩色 PNG，也不使用无关通用图标替代；图标和文字必须在 `NSStatusBarButton.effectiveAppearance` 下解析系统动态色，以匹配桌面自动选择的黑色或白色菜单栏前景。
- 今日总 token 在状态栏只显示垂直居中的紧凑数字，不增加 `T` 等冗余标签；完整口径保留在 tooltip 和可访问性文案。
- 今日总 token 使用系统菜单栏正文尺寸；5h/7d/mo 标签与重置截止时间使用系统前景色的辅助层级，但与背景的对比必须明显高于占位/禁用文字，并与主百分比保持可辨的主次色差。重置倒计时必须以“循环重置图标 + 紧凑时长”作为独立居中信息组，不展示裸的 `5d` 等易与额度周期混淆的文案；tooltip 和可访问性文案使用“5 天后重置 / resets in 5 days”完整语义。
- 状态栏宽度由模式、用户选择与服务端已确认的额度拓扑决定。数据刷新、位数变化和短暂不可用不得推动相邻菜单栏图标反复移动；只在一次成功读取确认单额度或双额度后更新布局。
- 服务端成功响应中不存在的额度窗口应自动隐藏，不保留空轨道或 `--`；加载、超时、解析失败和过期快照不能被误判为“当前无额度”，应保留上一次成功拓扑或显示明确状态。缺失数据不得伪造成 0。
- 设置页预览与真实状态栏必须共享同一展示模型和绘制器。

## 文案规范

- 标题使用结果导向文案，例如“用量趋势”“项目排行”，避免内部概念。
- 估算值必须带“估算”。
- 回退口径使用用户能理解的词，例如“线程口径”，避免孤立的“粗略”。
- 不使用夸张营销词，不做价值判断。
- 中英文界面应表达同一语义，不逐字机械翻译。

## 数据口径

- 精细统计来自本机 session token 事件。
- 线程口径来自本机线程记录，只能近似归因。
- API 等效价值、工具 token 分摊、预测值都属于估算。
- 官方额度、官方账单、本地估算必须清晰区分。
- 当数据不可用时，说明“记录不足”或“口径不可用”，不要伪造默认值。

## 隐私边界

- 默认只展示聚合指标、脱敏名称、路径尾名和状态。
- 完整本地路径只在确有帮助时通过 tooltip 展示。
- 不展示 prompt、回复正文、tool arguments、附件正文、auth、环境变量或 raw logs。
- 不新增遥测、远程分析、第三方上传或云同步。

## 可访问性

- 颜色不能是唯一信息通道，状态必须同时通过文字、图标或位置表达。
- 浅色、深色和高对比环境下保持可读。
- 可点击控件必须有明确视觉反馈和可理解的 tooltip 或 label。
- 小字号文本不能承担唯一关键含义。

## 维护规则

- 轻量与低资源占用高于装饰性效果：窗口静止且数据未变化时，不应持续触发逐帧重绘；窗口隐藏、最小化、完全遮挡或位于非活动桌面时，停止非必要轮询。
- 默认与省电动效模式只改变装饰性粒子的出现条件，不得影响额度信息的可见性、可读性或交互；悬停不能成为获取信息的必要条件。
- 后台定时任务必须设置合理容差，避免同频唤醒；高成本刷新不得并发执行，未变化的数据不得重复发布到界面。
- 新增组件先复用现有模式，再考虑抽象。
- 新增视觉 token 必须说明职责，不为单个场景临时造 token。
- 改动布局规范时，同步更新本文档。
- 设计规范只记录长期原则，不记录一次性诊断、迁移计划或临时实现细节。

## Palette Package 边界

- 应用只从签名包内的 `Resources/Palettes/<palette-id>/` 发现配色，不扫描用户目录、不执行脚本、不联网下载资源。
- 每个包必须同时提供浅色和深色 Token、本地化、来源与许可证；展示名称可以变化，稳定 ID 不得复用。
- View 只能读取解析后的 `EnvironmentValues.visualTokens`，不得直接读取 JSON、拼接资产路径或按 palette ID 写特例。
- SVG 只作为低频静态纹样，经过元素、引用、路径、体积和复杂度校验后由 AppKit 解码；进度百分比、端点位置和业务文字始终由通用组件计算。
- 默认包和社区包走同一加载链路。默认包失效时才使用编译内 `SafeDefaultVisualTokens`，确保应用仍可启动。
- 完整目录、字段、安全规则和验收命令见 [Palette Package v1](PALETTE_PACKAGES.md)。
